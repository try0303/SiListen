# SiListen Goal 模式执行文档

## 总目标

用连续八小时左右的集中执行，把 SiListen 从“功能能跑”推进到“日常可用、视觉稳定、交互可信”的状态。执行时以用户体验为第一优先级，避免为了追求炫技破坏现有稳定功能。

## 当前稳定基线

- 当前代码已经完成 `SiListenApp.kt` 拆分，主壳和页面实现不再混在一个超大文件中。
- 当前稳定提交为 `2645491 Split app UI shell and add roadmap`。
- 之前验证过 `.\gradlew.bat assembleDebug` 可以通过，应用可以安装并启动。
- 后续所有大改都必须基于小步提交推进，不能一次性重写大量核心交互。

## Goal 1：播放链路和迷你播放器稳定

### 目标

确保用户点歌后，底部迷你播放器稳定出现，并且能可靠进入播放页或歌词页。

### 执行步骤

1. 梳理当前播放状态来源，确认当前歌曲、播放状态、进度、封面、歌手名是否是单一可信数据源。
2. 检查迷你播放器显示条件，避免因为页面切换、歌单详情、搜索页、账号页导致播放器消失。
3. 优化迷你播放器布局，歌名和歌手不再被圆角或裁剪遮挡。
4. 给迷你播放器增加半透明、半模糊的玻璃质感，同时保证文字可读。
5. 点击迷你播放器进入播放页，播放页内再提供歌词、队列、评论等入口。

### 验收标准

- 任意页面点歌后，迷你播放器都能显示在底部栏上方。
- 切换首页、搜索、音乐库、账号后，迷你播放器不丢失。
- 点击迷你播放器能进入播放页。
- 歌名、歌手、播放按钮不被遮挡。

### 禁止事项

- 不允许为了修复布局问题删除迷你播放器。
- 不允许把迷你播放器藏到只有特定页面才显示。

## Goal 2：液态玻璃底部栏稳定

### 目标

让底部栏在点击、拖动、页面滑动三种操作下都准确、跟手，并尽量接近 KernelSU/Miuix 的液态玻璃质感。

### 执行步骤

1. 保持 `HorizontalPager` 作为页面切换的唯一状态源。
2. 点击底部栏时使用 pager 动画切换页面，不能手动维护另一套页面索引。
3. 页面滑动时让底部栏直接读取 pager 的偏移量，避免慢一拍。
4. 拖动底部栏指示器时根据实际拖动距离计算目标页面，不能出现首页点账号只到音乐库的问题。
5. 优化液态指示器绘制，让它有柔和的大小变化、透明感、边缘溢出感。
6. 浅色模式使用白灰透明背景，深色模式使用黑色透明背景。
7. 检查底部文字在玻璃背景下的可读性。

### 验收标准

- 首页点击搜索、音乐库、账号都能准确到达目标页。
- 搜索、音乐库、账号点击任意其他页面也能准确到达。
- 左右滑动页面时，底部液态指示器同步移动。
- 拖动底部栏时不跳页、不乱飞、不暴露方形矩形。
- 深色和浅色模式下底栏背景不会错色。

### 禁止事项

- 不允许再引入与 pager 状态竞争的独立 selectedIndex。
- 不允许删除液态玻璃功能来规避问题。
- 不允许出现“看起来能动但实际落点错一格”的交互。

## Goal 3：Apple Music 风格歌词页

### 目标

把歌词页打磨成 Android 平台的 Apple Music 风格沉浸式播放页。

### 执行步骤

1. 使用封面图作为背景，做高斯模糊并叠加深色渐变。
2. 歌词区域垂直居中，当前行固定在屏幕中线附近。
3. 当前歌词行使用白色、大字号、加粗、高对比显示。
4. 非当前歌词根据距离变小、变灰、变透明。
5. 播放进度变化时，歌词列表平滑滚动，当前行保持居中。
6. 实现逐字高亮填充时，只绘制一套歌词，避免重叠。
7. 点击歌词行跳转到对应时间点。
8. 底部进度条支持拖动，拖动时歌词同步跳转。

### 验收标准

- 当前歌词不会和逐字高亮文字重叠。
- 快进、拖动进度条、点击歌词行后，当前歌词能正确跳转。
- 背景沉浸但不影响歌词可读性。
- 页面无多余边框和杂乱控件。

### 禁止事项

- 不允许同时绘制完整歌词和逐字歌词造成重影。
- 不允许为了动画效果牺牲流畅性和可读性。

## Goal 4：网易云端内直连与登录

### 目标

减少对本地 API 服务的依赖，让普通用户不用开电脑端服务也能使用核心音乐能力。

### 执行步骤

1. 参考 `bujuan_music_api` 的接口思路，梳理可移植到 Kotlin 端内直连的请求。
2. 将本地 API 地址降级为调试或备用配置。
3. 登录二维码失败时，给出中文原因和下一步建议。
4. 需要打开确认时，优先尝试唤起网易云音乐 App。
5. 如果网易云接口变动导致登录受限，保留可替换的登录策略。

### 验收标准

- 普通使用不要求用户手动启动本地 API。
- 登录失败提示用户能看懂。
- 可唤起网易云 App 时不会默认打开网页。

### 禁止事项

- 不允许把开发者调试接口直接暴露给普通用户。
- 不允许用含糊提示掩盖登录失败原因。

## Goal 5：系统媒体通知控制

### 目标

通知栏必须表现为真正的音乐播放控制器，而不是普通通知。

### 执行步骤

1. 使用 Android 媒体通知样式和媒体会话。
2. 通知展示歌曲封面、歌名、歌手。
3. 通知动作包括上一首、播放或暂停、下一首、关闭。
4. 点击通知进入播放页或歌词页。
5. 播放状态变化时同步更新通知按钮状态。

### 验收标准

- 锁屏或通知栏中能看到标准音乐控制。
- 播放、暂停、上一首、下一首都能控制当前播放。
- 点击通知能回到应用内播放体验。

### 禁止事项

- 不允许用普通文本通知冒充媒体控制器。
- 不允许通知点击后进入无关页面。

## Goal 6：主题、设置和可读性

### 目标

让主题设置、Monet 开关、浅色深色模式、标题和列表文字都稳定可读。

### 执行步骤

1. 参考 KernelSU 的主题设置结构，统一主题选项和开关行为。
2. 修复 Monet 开关状态不准或文案不清楚的问题。
3. 设置页所有文案改成面向用户的中文。
4. 检查首页、歌单、今日播放、音乐库、本地音乐、账号、喜欢的音乐、我的歌单等标题颜色。
5. 确保浅色模式和深色模式都有明确的文本颜色层级。

### 验收标准

- 浅色模式下文字不会过浅。
- 深色模式下文字不会过暗。
- 主题设置文案清楚，用户知道每个选项影响什么。
- Monet 开关行为和显示状态一致。

### 禁止事项

- 不允许在用户界面显示内部开关名。
- 不允许只检查一种主题模式就结束。

## Goal 7：搜索入口和底部栏联动

### 目标

搜索入口要轻量、明显，并且与底部栏联动自然。

### 执行步骤

1. 将搜索按钮作为底部区域的独立浮动入口。
2. 点击搜索时，底部栏收缩到最近点击的页面状态，搜索栏展开。
3. 再次点击底部栏任意项时，搜索栏收起，底部栏恢复完整状态。
4. 保持底部栏液态动画不被搜索展开状态破坏。

### 验收标准

- 点击搜索后搜索体验展开自然。
- 点击底部栏后能退出搜索展开状态。
- 页面切换、底栏动画、搜索展开三者不会互相抢状态。

### 禁止事项

- 不允许搜索状态导致底部栏索引错乱。
- 不允许搜索展开后遮挡迷你播放器核心内容。

## Goal 8：测试、提交与推送规则

### 目标

建立不会反复回归的执行节奏，确保每个阶段都有清晰的稳定点。

### 执行步骤

1. 每个 Goal 完成后先查看 `git status --short`。
2. Kotlin 改动必须运行 `.\gradlew.bat assembleDebug`。
3. 高风险交互改动必须安装到设备并检查崩溃日志。
4. 验证通过后再本地提交。
5. 推送前必须等待本人实际检测并确认无误。

### 验收标准

- 每个阶段都有可回退的提交。
- 没有经过用户实际确认，不推送远端。
- 出现回归时能快速定位到具体提交。

### 禁止事项

- 不允许未经确认直接推送。
- 不允许把多个无关大改塞进同一个提交。
- 不允许为了赶进度跳过构建验证。

## 推荐执行顺序

1. 先稳定播放链路和迷你播放器。
2. 再稳定液态玻璃底部栏和页面联动。
3. 然后集中打磨 Apple Music 风格歌词页。
4. 接着完善网易云端内直连和登录提示。
5. 再完善系统媒体通知控制器。
6. 最后统一主题设置、浅色深色可读性和用户文案。

## 每轮执行输出格式

每轮完成后用中文输出：

- 本轮完成了什么。
- 修改了哪些核心文件。
- 运行了哪些验证。
- 是否有未解决风险。
- 是否已经提交。
- 是否等待本人检测后再推送。
